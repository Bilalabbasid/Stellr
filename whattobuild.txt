```sql
-- Restaurants table
-- Each row is one restaurant we manage
create table restaurants (
  id uuid primary key default gen_random_uuid(),
  slug text unique not null,
  -- slug is what appears in the URL: stellr.biz/r/[slug]
  -- must be lowercase, no spaces, use hyphens e.g. chaaye-khana
  name text not null,
  -- the restaurant's public display name shown to customers
  logo_url text,
  -- full URL to their logo, hosted anywhere, shown on their funnel page
  google_review_url text not null,
  -- the direct Google review link where happy customers get redirected
  -- format: https://search.google.com/local/writereview?placeid=XXXXX
  owner_name text,
  owner_email text,
  owner_whatsapp text,
  -- owner_whatsapp in international format e.g. 923001234567
  active boolean default true,
  -- if false, the funnel page shows a generic message instead
  created_at timestamp with time zone default now()
);

-- Feedback table
-- Every customer interaction is stored here whether positive or negative
create table feedback (
  id uuid primary key default gen_random_uuid(),
  restaurant_id uuid references restaurants(id) on delete cascade,
  rating text not null,
  -- values: excellent, good, okay, poor, terrible
  redirected_to_google boolean default false,
  -- true if the customer was sent to Google reviews
  complaint_text text,
  -- only filled if rating is okay, poor, or terrible
  complaint_tags text[],
  -- array of selected tags from: food_quality, staff_behaviour,
  -- service_speed, cleanliness, pricing, other
  customer_ip text,
  -- store anonymised IP for basic spam prevention
  created_at timestamp with time zone default now()
);

-- Index for fast lookups by restaurant and date
create index on feedback (restaurant_id, created_at desc);
create index on restaurants (slug);

-- Row level security
alter table restaurants enable row level security;
alter table feedback enable row level security;

-- Anyone can read restaurant data so the funnel page loads without auth
create policy "Public read restaurants"
  on restaurants for select using (true);

-- Anyone can submit feedback so customers do not need to log in
create policy "Public insert feedback"
  on feedback for insert with check (true);

-- Anyone can read feedback for the dashboard
-- In production you would restrict this but for now keep it open
create policy "Public read feedback"
  on feedback for select using (true);
```

**STEP 2 — DYNAMIC RESTAURANT FUNNEL PAGE**

Create: app/r/[slug]/page.jsx

This is the most important page. Every customer lands here after scanning the QR code. Here is exactly what it must do:

When the page loads use the slug from the URL to fetch the matching restaurant from Supabase. This should be a server component fetch so the page loads fast with no loading spinner for the customer.

If the restaurant is found render the funnel in stages using React state. The page has three stages: rating stage, complaint stage, and thank you stage.

**Rating stage:**
Show the restaurant's logo centered at the top if logo_url exists. Below it show their name in large bold text. Below that show the question "How was your experience at [name] today?" in clear readable text. Below that show five buttons stacked vertically each taking full width and easy to tap on a phone. The buttons are: Excellent, Good, It was okay, Not great, and Terrible. Style them cleanly with the restaurant's page feeling premium and trustworthy, not cheap.

When the customer taps Excellent or Good: save a feedback row to Supabase with the restaurant_id, the rating value, and redirected_to_google set to true. Then immediately redirect the customer to the restaurant's google_review_url. Open it in the same tab so the Google review page takes over.

When the customer taps It was okay, Not great, or Terrible: save a feedback row to Supabase with the rating and redirected_to_google as false. Then move to the complaint stage without reloading the page.

**Complaint stage:**
Show a heading that says "We are sorry to hear that. Please tell us what happened so we can make it right." Below that show a text area where they can write their complaint. It should be optional not required. Below that show checkboxes so they can select what the complaint is about. The checkbox options are: Food Quality, Staff Behaviour, Service Speed, Cleanliness, Pricing, Something else. Below that show a single submit button. When submitted update the feedback row in Supabase with the complaint_text and complaint_tags array. Move to the thank you stage.

**Thank you stage:**
Show a simple warm message: "Thank you for your feedback. We have passed this on to the team and they will look into it." Show the restaurant logo again if it exists. Show stellr.biz in small text at the very bottom.

The entire page must be fully mobile optimised. Large tap targets, readable font sizes, no horizontal scrolling, centered layout with max width of 480px and auto margins.
